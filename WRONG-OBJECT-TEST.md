# Wrong Object Test

## Public threshold

Most people do not need a theory of totality.

They need to see one thing:

the decision was rational only because it was made on the wrong object.

OMNIA begins there.

The Wrong Object Test is the public operational form of OMNIA.

It does not start with philosophy.

It starts with a concrete failure:

a correct calculation, answer, metric, model, or decision is applied to a reduced object instead of the real structure that produced the problem.

The result can be locally correct and still structurally wrong.

## Core sentence

OMNIA detects when a correct answer is answering the wrong object.

## Minimal formula

Let:

T = accessible generative structure or wider field

pi(T) = reduced visible projection of T

D = decision, inference, calculation, classification, or judgement

The structural error appears when:

D(pi(T)) is treated as D(T)

In compact form:

E(T, pi) = Assert(pi(T) = T)

Operationally:

E_D(T, pi) = Delta(D(pi(T)), D(T))

If the decision changes when the missing field is restored, the original decision was made on the wrong object.

## The test

Ask five questions.

1. What object is being used for the decision?

2. Is that object the full accessible structure, or only a projection?

3. What was removed, hidden, averaged, simplified, renamed, or made invisible?

4. Does the decision change when the missing structure is restored?

5. If the decision changes, what false totalization produced the error?

If the answer changes after field restoration, the problem was not only a wrong answer.

It was the wrong object.

## Why this matters

Many systems do not fail because people cannot calculate.

They fail because people calculate correctly on the wrong object.

The number is correct.

The model passes.

The metric improves.

The answer sounds coherent.

The test is passed.

The aggregate is true.

The prompt is answered.

The representation is valid.

But the object being used is not the real structure.

That is the central OMNIA failure.

## Case 1 — Monty Hall

Wrong object:

two closed doors remain.

Real structure:

initial choice, car placement, host knowledge, host rule, constrained opening, conditional information.

Wrong decision:

treat the remaining two doors as symmetrical.

Corrected decision:

staying wins with probability 1/3.

switching wins with probability 2/3.

Diagnosis:

the terminal visible projection was treated as the full stochastic model.

## Case 2 — Simpson's Paradox

Wrong object:

the aggregate success rate.

Real structure:

the conditioned subgroup structure that produced the aggregate.

Wrong decision:

choose the treatment that wins in the aggregate.

Corrected decision:

after restoring the hidden conditioning variable, the opposite treatment can win in every relevant subgroup.

Diagnosis:

the aggregate was treated as the causal decision object.

## Case 3 — AI frame collapse

Wrong object:

the words preserved in the answer.

Real structure:

the originating frame that gives those words their direction and meaning.

Wrong decision:

the answer is accepted because it appears to follow the topic.

Corrected decision:

the answer is rejected if it preserves vocabulary but replaces the originating field.

Diagnosis:

word preservation was treated as frame fidelity.

## Case 4 — Numerical representation

Wrong object:

a digit string in one base.

Real structure:

the numerical object that appears through finite base representations without coinciding with any of them.

Wrong decision:

identify the number with its familiar notation.

Corrected decision:

use the representation while refusing to treat it as the object.

Diagnosis:

a finite base-expression was treated as the number.

## Case 5 — Local correctness

Wrong object:

the answer that passed the immediate local test.

Real structure:

the wider field in which the answer must remain stable.

Wrong decision:

accept the answer as valid because it is locally correct.

Corrected decision:

test whether the answer survives reframing, dependency restoration, missing constraints, and future conditions.

Diagnosis:

local correctness was treated as total stability.

## Public translation

Do not say first:

the finite projection is not the totality.

Say:

you are solving the wrong object.

Then show the reversal.

Before OMNIA:

this answer is correct.

After OMNIA:

correct for what object?

Before OMNIA:

this number is true.

After OMNIA:

true inside which structure?

Before OMNIA:

the model passed.

After OMNIA:

passed under which missing field?

Before OMNIA:

the system works.

After OMNIA:

works until which excluded condition returns?

## Operational rule

Never ask only:

Is it correct?

Ask:

What object is it correct about?

If the object is only a projection, the answer may be locally valid but structurally unsafe.

a decision becomes structurally wrong when it is made on a projection while being used as if it were made on the full accessible structure.
