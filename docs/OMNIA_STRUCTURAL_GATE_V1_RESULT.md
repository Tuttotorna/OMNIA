# OMNIA STRUCTURAL GATE V1 — RESULT

## Purpose

This experiment represents the first transition from:

structural observation → structural intervention

A minimal gate is applied to real LLM outputs.

---

## Input

data/real_structural_dataset_v1.jsonl

- 60 real outputs
- generated by a local model (flan-t5-small)
- no manual curation

---

## Structural Classes

Each output is classified as:

- atomic  (length ≤ 2)
- short   (length ≤ 8)
- long    (length > 8)

---

## Gate Logic

atomic → REJECT

short: malformed > 0 OR symbol_density > 0.15 → REVIEW else → PASS

long: digit_density > 0.20 OR symbol_density > 0.20 OR malformed > 3 → REVIEW else → PASS

---

## Results

### Gate distribution

PASS   = 24 (40.0%) REVIEW = 28 (46.7%) REJECT =  8 (13.3%)

### Structural distribution

atomic =  8 (13.3%) short  = 17 (28.3%) long   = 35 (58.3%)

---

## Interpretation

### 1. Gate is balanced

not too permissive not too restrictive

- PASS ≈ 40%
- REVIEW ≈ 47%
- REJECT ≈ 13%

This is a usable operating point.

---

### 2. Atomic collapse is controlled

All atomic outputs are correctly mapped to:

REJECT

This validates the base rule.

---

### 3. Long drift is partially captured

Most real outputs fall into:

long regime

The gate does not reject them blindly, but:

filters unstable ones → REVIEW lets stable ones pass → PASS

---

### 4. REVIEW is the dominant signal

REVIEW ≈ 47%

This is expected.

It means:

the system detects structural uncertainty

Not just failure.

---

## Core Insight

LLM outputs are not binary (correct / incorrect)

They exist in structural regimes:

stable

unstable

collapsed


The gate exposes this.

---

structural stability
