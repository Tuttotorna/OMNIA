# Wrong Object Test Matrix

## Purpose

This matrix turns OMNIA into a public operational test.

The goal is simple:

show where a decision, answer, metric, or model is using the wrong object.

## Matrix

| Case | Object used | Missing structure | False totalization | Decision before | Decision after |
|---|---|---|---|---|---|
| Monty Hall | Two closed doors | Initial choice, host knowledge, host rule, conditional information | Terminal projection treated as full stochastic model | 50/50 | Switch wins 2/3 |
| Simpson's Paradox | Aggregate success rate | Subgroup or causal structure | Aggregate treated as causal decision object | Choose aggregate winner | Choose structure-correct winner |
| AI frame collapse | Preserved words | Originating frame | Words treated as frame fidelity | Accept fluent answer | Reject frame-betraying answer |
| Numerical representation | Digit string or base-expression | Numerical object beyond finite representation | Representation treated as object | Identify notation with number | Use notation without sovereignty |
| Local correctness | Answer passing local test | Wider stability field | Correctness treated as stability | Accept correct answer | Test field stability |
| KPI failure | Performance metric | System dependencies and hidden costs | Metric treated as system | Optimize KPI | Evaluate structural effect |
| Security failure | No visible incident | Threat surface, latent vulnerability, adversarial field | Absence of incident treated as security | Declare safe | Test attack surface |
| Policy failure | Single public statistic | Distribution, incentives, side effects, long-term field | Statistic treated as population reality | Legislate on one number | Restore system context |
| Economic failure | Revenue or GDP movement | Destruction, debt, externalities, future fragility | Movement of money treated as value creation | Call it growth | Separate growth from damage |
| Project failure | Contact or idea | Confirmation, roles, logistics, dependencies | Contact treated as structure | Assume project is moving | Build confirmed operational map |

## Minimal diagnostic formula

T = accessible total or generative field

pi(T) = reduced projection

D = decision or inference

Wrong Object Error:

E_D(T, pi) = Delta(D(pi(T)), D(T))

If D(pi(T)) differs from D(T), the original decision was made on the wrong object.

## Reading rule

The point is not that every projection is bad.

Every finite system needs projections.

The error begins only when the projection is used as if it were the complete object.

Projection is allowed.

False totalization is not.

## Public wording

Instead of saying:

you ignored totality.

Say:

you used the wrong object.

Instead of saying:

the fragment is not the whole.

Say:

your answer is correct only for the reduced object.

Instead of saying:

the field was excluded.

Say:

the missing structure changes the decision.

## Relation to OMNIA

OMNIA is not a theory that replaces specialized knowledge.

OMNIA is a diagnostic layer that asks whether the specialized answer is attached to the correct object.

A statistician may calculate correctly.

A doctor may classify correctly.

An AI may answer coherently.

A manager may optimize correctly.

A model may validate correctly.

But OMNIA asks:

correct for which object?

That is the public door.

## Final sentence

The world does not need to hear first that the fragment is not the whole.

The world needs to see that the decision changes when the wrong object is replaced by the real structure.\n
